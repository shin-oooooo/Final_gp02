## Fig4.1 · 论点

资产分类在极端环境下的集体失效，可由 **「模型—模型应力检验」** 与 **「模型应力—市场载荷方向检验」**
**提前 1–5 个交易日**识别。由于（i）测试窗长度有限；（ii）研究事件（如地缘冲突）的外生性较强且
具有持续性，我们将测试窗内的「市场逻辑断裂」假设为 **至多发生一次**。当两路独立检验同时给出
有效预警时，**优先采用「模型应力—市场载荷方向」检验的告警日作为最终预警日**——其原因为：
模型—市场检验直接刻画**叙事 / 载荷方向相对于价格轨迹的偏离**，更贴近真实市场的非稳态信号；
模型—模型应力（JSD）只反映预测分布层面的分歧，**不直接对应已实现的市场状态变化**。

## Fig4.1 · 情形 B · 仅「模型—模型」成功（⚠ 参考价值较低）

- **模型—模型**：`{mm_verdict}`，告警日 `{mm_t0_date}`
- **模型—市场**：`{mv_verdict}`

> ⚠ **参考价值较低提示**：仅模型间分歧（JSD）触发，而叙事 / 载荷方向并未与价格轨迹背离。
> 这意味着**预测层面的不确定性上升**，但并不能直接等价于**市场层面的真实失效**；在以
> `**{mm_t0_date}`** 作为参考预警日时，必须同时报告本提示。

**结论**：以 `**{mm_t0_date}`** 作为参考预警日。模型层出现显著的预测分布分歧
（JSD 应力越基线），表明三路结构模型对未来收益分布形成不可忽略的不一致；
但语义—数值滚动余弦在同期仍 ≥ 0，说明**叙事维度尚未与价格轨迹产生方向性背离**。
因此该信号本身**不足以独立支撑「市场逻辑断裂」的判定**；结论应谨慎，并附「参考价值较低」。

### 进一步排查建议（按优先级）

1. **逐模型诊断**：用 Phase2 留存的 `prob_nll_mean` / `prob_dm_pvalue_vs_naive` / `prob_coverage_95`
  审视 ARIMA / LightGBM / Kronos 三路的 OOS 表现：
  - 若**单一模型** NLL 明显恶化、DM 指向其显著劣于 Naive，则 JSD 升高多来自该模型的
  **参数漂移**；对应补救是**触发再训练 / 调整 `oos_fit_steps`**。
  - 若**三模型同向恶化**，则更可能是**输入特征的协变量偏移**；对应补救是**复核训练窗起止**、
  **扩大训练池**或**引入更强正则**。
2. **训练—测试分布一致性**：在告警日 `**{mm_t0_date}`** 前 20–40 个交易日做 KS / Wasserstein
  检验，量化收益序列边际分布的结构变化；若显著拒绝同分布原假设，应收紧 `train_end`
   或重训模型。
3. **叙事缺位复核**：对照同期 S_t（情绪）与新闻面，判断是否存在「事件已发生但尚未被
  文本维度向量化」。若确有遗漏关键词（例如突发性政策 / 地缘事件的新领域词汇），应把相应
   模板加入 `sentiment_proxy` 的关键词集合后再重跑余弦，再判定本情形的结论是否需要修正。
4. **统计纪律**：在以上三步未给出阳性证据前，**不应**临时下调 `verify_std_quantile_pct` /
  `verify_crash_quantile_pct` / `verify_tail_quantile_pct` 去强制让模型—市场检验通过——
   这将破坏答辩层面的检验独立性与可重复性。

## Fig4.1 · 情形 C · 仅「模型—市场」成功

- **模型—模型**：`{mm_verdict}`
- **模型—市场**：`{mv_verdict}`，告警日 `{mv_t0_date}`

**结论**：以 `**{mv_t0_date}`** 作为最终预警日。语义载荷方向与数值轨迹出现持续背离
（滚动余弦 < 0），表明**叙事维度先于价格 / 波动反映出结构性变化**；这是市场层面真实失效的
强提示，参考价值高。模型—模型应力（JSD）此刻未越基线，可能由于：

- 三路结构模型对「新信号」**尚未形成共识性的分歧**（多见于外生事件初期，分歧效应滞后）；
- 模型协方差结构同质化较强，分布层面的分歧被低估。

### 操作建议

1. **依据市场—载荷检验作出防御决策**：在 `{mv_t0_date}` 后的 5 个交易日内监控大跌 / 厚尾
  占比是否持续高于训练基线；若持续，应触发 Phase3 的防御等级重评。
2. **同步追踪 JSD 是否滞后命中**：在 `{mv_t0_date}` 之后的 3–10 个交易日继续采集 JSD
  序列，若延迟越基线则属于**两路同向但时间错位**的情况，可作为方法论稳定性证据。
3. **补强信号关联性证据**：检查 Phase2 `cosine_semantic_numeric` 序列在 `{mv_t0_date}`
  附近是否单调下降；若呈现陡降并伴随负值持续 ≥ 2 日，则可视为强证据。

## Fig4.1 · 情形 D · 两路均预警失败（归因）

- **模型—模型**：`{mm_verdict}`
- **模型—市场**：`{mv_verdict}`

**结论**：两路检验均未在 1–5 个交易日窗口内给出有效预警。该失败可能源自以下因素，
按"先排查方法论 / 数据，再考虑模型 / 阈值"的顺序处理：

### 可能原因（按优先级排序）

1. **测试窗未覆盖真实断裂日**：测试长度或采样频率不足以捕获目标事件；建议核查
  `test_start` / `test_end` 是否完整覆盖目标事件窗。
2. **训练窗污染**：训练窗包含结构变化段（如 2024 H2 之后）会令训练基线本身被「污染」，
  阈值随之放宽，使任何后续的样本外异常都难以越过。建议重置训练窗到结构稳态段并重训。
3. **阈值过保守**：训练窗分位（`verify_std_quantile_pct` / `verify_crash_quantile_pct` /
  `verify_tail_quantile_pct`）过高，导致信号在实际发生前不触发。
4. **模型同质性过强 → JSD 退化**：三路结构模型对新信号反应相似，分歧效应被压制；
  建议引入互补模型（如外加非线性核或纯统计模型），以提升模型间分歧度的灵敏度。
5. **语义代理失配**：情绪 / 载荷信号维度与目标事件的因果链较弱（例如外生政策事件未被文本
  向量化）。建议复核 `sentiment_proxy` 的关键词集合并补充事件领域词典。

### 操作建议

- 优先复核「测试窗起止 + 训练窗起止 + 标的池」三件套；这三处任一错配都会同时压制两路检验。
- 仅在以上排查完成且仍未命中的情况下，再考虑放宽 `verify_*_quantile_pct`。
- 若仍无显著性，转而检查「方法论假设」本身：测试窗内是否真有结构性断裂事件？
若数据上无可识别的断裂，本结果应被解读为**预警系统对当期窗的稳态判定正确**，
而非「预警失败」。

## Fig4.2 · 情形 C · 策略构建成功（最优）

`defense_level ≥ 1`、`|test_mdd_pct_cvar|` 在三条曲线中最小、`test_terminal_cumret_cvar`  
同为最高。Adaptive Optimizer 既压缩了回撤，又维持了  
累计收益。此检验证明，**“逻辑熔断”**（即识别到失效条件后自动切换至最小化 CVaR 策略）是应对黑天鹅事件的机械式必然选择，它通过牺牲平稳期的微小超额收益，换取了极端环境下的生存概率。

## Fig4.2 · 情形 D · 策略构建成功（主目标达成）

`defense_level ≥ 1`、`|test_mdd_pct_cvar|` 最小、`test_terminal_cumret_cvar` 非

## Fig4.2 · 情形 E · 策略构建失败

`defense_level ≥ 1` 且 `|test_mdd_pct_cvar|` 非最小。即便 `test_terminal_cumret_cvar`
在某子情形中最高，也未达到"防御即压缩尾部风险"的基本目标。

### E-1 · `|MDD_cvar|` 非最小 且 `cumret_cvar` 最高

等价于"Adaptive Optimizer 在压力下反而变得激进"：

1. **CVaR 估计样本不足**：测试窗左尾样本过少，经验 CVaR 偏乐观，导致优化器得到的
  `w_cvar` 被"高估有效性"；建议复核 `cvar_alpha`（默认 0.05）是否过宽，或延长
   `mc_horizon_days`。
2. **协方差估计退化**：Phase 0 的 Σ 若被少量异常日主导，Min-CVaR 优化会把权重集中
  到"历史看似安全、尾部实则偏厚"的标的。
3. **防御等级误升**：`resolve_defense_level` 在当期不应升至 Level 2；核查
  `jsd_stress` / `pseudo_melt` / `prob_full_pipeline_failure` 是否存在假阳性。

### E-2 · `|MDD_cvar|` 非最小 且 `cumret_cvar` 非最高

最严重子情形，Adaptive Optimizer **既未护住回撤，也未维持收益**：

1. **测试窗与训练窗存在结构性断裂**：训练窗的 Σ 在测试窗已失真，`w_cvar` 在样本外
  整体劣于两基准；补救为重置 `train_end`、重训，或采用更短估计窗 + 递归更新。
2. `**blocked_symbols` 拦截面过大**：Phase 1 过度拦截（例如 ADF 阈值过严）使可用
  持仓面萎缩，红线因剩余标的集中而同时恶化 MDD 与收益；核查 `pol.tau_vol_melt` /
   `pol.tau_return_ac1`。
3. **语义先验方向错误**：Level 1 下 λ·负面语义惩罚若与实际叙事反向（情绪分极性颠倒），
  会把权重推向错误方向；核查 `vader_st_series_partition_cumulative_from_detail` 输出的
   S_t 极性与新闻面一致性。
4. **目标函数与约束不兼容**：`w_cvar` 若命中持仓上/下限，实际为"约束解"而非优化解，
  Adaptive Optimizer 自由度在该窗被压缩至零。

### Fig 4.2 方法局限性

- **测试窗长度**：测试窗越短，MDD 随机性越大；`|MDD%|` 的大小关系可能被一两日极端收益
翻转。建议以 `comparison_active=True` 下的 MC 反事实为主判据。
- **"构建成功" ≠ "长期稳健"**：Fig 4.2 仅能论证在**该测试窗**内 Adaptive Optimizer 压缩
了已实现 MDD；跨窗稳定性需额外的滚动回测（非本页职责）。
- **MC 反事实的种子依赖**：尽管使用相同随机种子，不同跳跃强度 / 情景注入下反事实的
"分位点"对种子仍有可察觉的偏差；应关注趋势性结论而非单次百分点。
