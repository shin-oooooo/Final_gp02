# Sidebar-Left · 参数问号展开解释

> 每个参数旁「?」按钮点击后展开的**详细说明**。
>
> - 以 `tau_`* 等命名的键挂在 **单参数小标题** 右侧（防御等级四个 τ）。
> - 以 `block_`* 命名的键挂在 **分组子标题** 右侧（JSD 应力 / 可信度 / 影子测试 / 模型预测 / Level 1 λ / 双轨蒙特卡洛）。

# ── 小标题级（防御等级四个 τ）──

tau_h1: "τ_H1 越大，对资产抗风险能力较弱的判断越严格。结构熵低于 τ_H1 时，视为投资组合相对同质化、抗风险能力弱，防御等级切换到 Level 1（警戒）。"

tau_l2_l1: "可信度评分衡量四个数值模型预测结果的可信程度。τ_L2 与 τ_L1 越小，对模型「集体失效」的判断越严格。可信度评分在 τ_L2 与 τ_L1 之间时认为模型可信度较低，防御等级切换到 Level 1（警戒）；低于 τ_L2 时认为模型可信度低，防御等级切换到 Level 2（熔断）。"

tau_vol: "τ_vol 越小，对资产「伪平稳」的判断越严格。年化波动低于 τ_vol 时，资产被视为「伪平稳」资产，环境变化后可能呈现剧烈浮动，防御等级切换到 Level 1（警戒）。"

tau_ac1: "τ_AC1 越大，对资产「预测规律不足」的判断越严格。一阶自相关系数高于 τ_AC1 时，视为投资策略的逻辑有失效的风险，防御等级切换到 Level 1（警戒）。"

# ── 子标题级（批注于子标题；组内控件不再各自带问号）──

block_jsd_stress: "JSD 散度是衡量模型分歧程度的重要指标；通过在时间轴上动态对比三模型 JSD 散度均值是否超过基线，系统判断模型是否「集体失效」，并预测模型何时「集体失效」。k_JSD 是基线的放大倍数——k_JSD 越高，基线越高，对模型分歧的判断越严格。ε 给基线一个正的数值下界，避免在极端情况下基线值接近 0，导致判定不稳定或失真。"

block_credibility: "可信度评分 = 基准项 − 惩罚项。基准项系数越高，可信度对模型分歧越迟钝；惩罚项系数越高，可信度对模型的预测精确度变化越敏感；惩罚上限防止对可信度评分的过度惩罚。"

block_shadow: "「影子测试」负责选出每个标的表现最佳的模型。打分时，α 越接近 1 越看点预测精度（MSE），越接近 0 越看概率预测精度（JSD)。样本（训练集）尾部特定时间长度的数据将作为伪样本外测试集，用来计算MSE和JSD值。时间长度越短，测试速度越快，结果精度越低。"

block_model_predict: "在正式测试集里，在「当时刻之前」的信息上重新拟合一组模型参数的次数。重算次数多，测试速度越快，结果精度越低。"

block_lambda: "防御等级为 Level 1 时，Adaptive Optimizer 的优化目标为最小化一个加和：年化波动与「负面情绪暴露」的罚分。λ 越大，对「负面情绪暴露」的罚分越多，在优化权重时系统对负面罚分越敏感。"

block_mc: "「双轨蒙特卡洛」测试 Adaptive Optimizer 优化的资产组合策略在普通行情和极端情况下的表现，可自注入黑天鹅事件以模拟「市场逻辑断裂」。"
